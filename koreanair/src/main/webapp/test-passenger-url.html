승객 수 URL 파라미터 테스트

아래 링크를 클릭해서 booking.jsp에서 승객 수가 올바르게 파싱되는지 확인하세요.

성인 2명
URL: passengers=성인 2명
 성인 2명 (+ 인코딩)
URL: passengers=성인+2명
 성인 2명 (URL 인코딩)
URL: passengers=%EC%84%B1%EC%9D%B8%202%EB%AA%85
 성인 2명, 소아 1명
URL: passengers=성인 2명, 소아 1명
 성인 2명, 소아 1명 (+ 인코딩)
URL: passengers=성인+2명,+소아+1명
 성인 1명, 소아 1명, 유아 1명
URL: passengers=성인 1명, 소아 1명, 유아 1명

브라우저 개발자 도구 확인 방법

  1. 위 링크 중 하나를 클릭
  2. booking.jsp 페이지에서 F12를 눌러 개발자 도구 열기
  3. Console 탭에서 승객 파싱 결과 확인
  4. 페이지에 생성된 승객 정보 카드 개수 확인

서버 로그 확인 방법

서버 콘솔이나 로그 파일에서 다음과 같은 메시지를 확인하세요:

=== 승객 정보 파싱 ===
원본 passengers 파라미터: 성인+2명
URL 쿼리 스트링: passengers=성인+2명
디코딩 후 passengers 파라미터: 성인 2명
성인 매칭됨: 성인 2명 -> 2
최종 결과 - 성인: 2명, 소아: 0명, 유아: 0명
총 승객 수: 2
성인 승객 폼 생성 시작 (adultCount=2)
